1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
NewGenPrep is an AI-driven, end-to-end technical recruitment platform designed to 
replace traditional screening with high-fidelity, adaptive AI interviews. It 
specializes in transitioning from static DSA questions to "debug and build" 
engineering challenges (Round 2), mimicking real-world senior pair programming.

--------------------------------------------------------------------------------
2. TECHNOLOGY STACK
--------------------------------------------------------------------------------
FRONTEND:
- Framework: Next.js 14/15 (App Router)
- Language: TypeScript
- Styling: Vanilla CSS + Tailwind CSS (Adaptive Design)
- Components: Radix UI / Shadcn UI
- Code Editor: Monaco Editor (@monaco-editor/react)
- State Mgt: React Hooks + Session Persistence

BACKEND:
- Framework: FastAPI (Python 3.10+)
- Async Engine: ASGI (Uvicorn)
- Security: JWT Authentication, NoSQL Injection Protection, XSS Sanitization
- Database Driver: Motor (Async MongoDB Driver)

DATABASE:
- Provider: MongoDB Atlas
- Nature: Document-based (NoSQL) for flexible interview session storage

AI & EXECUTION:
- LLM Provider: Azure OpenAI (GPT-4o-mini / o4-mini)
- Audio Processing: Azure AI Whisper (Speech-to-Text)
- Vision: AI-based Proctoring Analysis
- Code Sandbox: Judge0 Community Edition (High-Speed Execution API)

--------------------------------------------------------------------------------
3. SYSTEM ARCHITECTURE
--------------------------------------------------------------------------------
Modular micro-services architecture with clear separation of concerns:
- Frontend: Handles UI, Monaco state, and real-time candidate signals.
- Router Layer: FastAPI routes for Auth, Interview, HR, and Execution.
- Service Layer: AI Engine (ai_engine.py), Report Engine (report_engine.py), 
  Security Layer (security.py).
- Persistence: MongoDB for users, candidate profiles, and session outcomes.
  
--------------------------------------------------------------------------------

[IMPLEMENTED]

- ROUND 2 (TECHNICAL): Live IDE with Monaco, Judge0 sandbox, and AI pair-programming hints.

- HR DASHBOARD: Candidate comparison, real-time analytics aggregation (FIX 1).
- REPORT ENGINE: Advanced HTML report generation with per-question breakdown and download (FIX 3).
- SCHEDULING SYSTEM: HR scheduling with automated 24h and 1h reminders (FIX 2).
- RUBRICS & EVALUATION: HR-defined scoring criteria for consistent grading.

[PLANNED / IN-PROGRESS]
- ADVANCED AI PAIRING: Contextual follow-up questions based on specific

5. DATA MODELS (DATABASE ENTITIES)
--------------------------------------------------------------------------------
- USERS: {id, email, password_hash, role: "candidate"|"hr", name}
- SESSIONS: {session_id, user_id, status, current_stage, responses: [], 
             questions_asked: [], proctoring_warnings: []}
- CANDIDATES: {id, name, email, phone, professional_summary, resume_data, 
               interview_history: []}
- QUESTION_SETS: {id, category, questions: [{text, difficulty, tags}]}
- SCHEDULES: {id, candidate_name, time, status, duration}
- RUBRICS: {id, name, competencies: [{name, weight, description}]}
