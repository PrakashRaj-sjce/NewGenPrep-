OVERVIEW
--------
  Implemented a comprehensive backend optimization and security layer to protect
  NewGenPrep from malicious payloads, ensure stable memory usage, and provide
  structured logging.


================================================================================
OPTIMIZATION 1: NOSQL INJECTION PROTECTION
================================================================================

  PROBLEM:
    FastAPI passes JSON payloads directly as Pydantic models. If dictionaries
    or arrays contain MongoDB operators (e.g., $gt, $ne, $where), attackers
    could manipulate queries inside services if standard validation is bypassed.

  SOLUTION:
    Created a recursive `check_nosql_injection` function inside `security.py`.
    It traverses all incoming nested JSON payloads and aggressively blocks
    any keys or string values starting with MongoDB operator prefixes like `$`.

  APPLIED TO:
    - Auth Routes (`/register`, `/login`, `/reset-password`, `/forgot-password`)
    - Interview Routes (`/interview/start`, `/interview/respond`)
    - HR Routes (`/hr/questions`, `/hr/settings`, `/hr/schedule`, `/hr/rubrics`)


================================================================================
OPTIMIZATION 2: FILE UPLOAD HARDENING (MAGIC BYTES)
================================================================================

  PROBLEM:
    Relying purely on file extensions for uploads limits security.
    An attacker could rename a malicious executable (`virus.exe`) to `resume.pdf`
    and upload it, potentially compromising servers processing those files.

  SOLUTION:
    Created `validate_file_upload(filename, bytes, type)` in `security.py`.
    It validates:
      1. Magic Bytes Check: Ensures `resume.pdf` actually begins with `%PDF` memory footprint.
      2. Path Traversal blocks (`../` injections in filenames).
      3. Global Size Limit assertion before proceeding.
      4. Hard blocklist for dangerous extensions (`.exe`, `.bat`, `.php`, etc.).

  APPLIED TO:
    - `/api/transcribe` (Audio validation)
    - `/api/interview/upload-resume` (PDF extraction pipeline)


================================================================================
OPTIMIZATION 3: XSS & INPUT SANITIZATION
================================================================================

  PROBLEM:
    User inputs in string fields (like Category, Name, Resume Name) can contain
    malicious `<script>` tags that fire when rendered on the HR Dashboard.

  SOLUTION:
    Created `check_xss` and `sanitize_string` functions.
    Strips raw null bytes, identifies typical DOM injection patterns (javascript:,
    onloads=, eval(), iframe), and asserts strict content bounds.

  APPLIED TO:
    - All HR settings strings and schedule creation (Candidate Name)
    - `validate_name`, `validate_email`, `validate_password`


================================================================================
OPTIMIZATION 4: REQUEST TRACE LOGGING
================================================================================

  PROBLEM:
    Standard Uvicorn logs do not track the precise lifecycle and duration of nested
    asynchronous operations for individual requests. Debugging production bottlenecks
    is difficult.

  SOLUTION:
    Implemented `RequestLoggingMiddleware` in the starlette pipeline.
    - Generates a unique `X-Request-ID` for every network call.
    - Computes accurate response latency (milliseconds).
    - Excludes health check noise.
    - Maps 5xx errors directly to the incoming request payload.


================================================================================
OPTIMIZATION 5: PAYLOAD MASS-LIMITING
================================================================================

  PROBLEM:
    DDoS vector: Attackers sending 2GB JSON bodies could crash the Uvicorn workers
    through memory exhaustion (OOM), taking down the platform.

  SOLUTION:
    Implemented `RequestSizeLimitMiddleware` configured to 50MB.
    Immediately rejects massive payloads at the ASGI boundary (413 Payload Too Large)
    before standard routing execution begins.
