OPTIMIZATION 1: SPA ROUTING FIXED (back/forward navigation)
================================================================================

  PROBLEM:
    next.config.mjs had output: export which forces static HTML export.
    Static export has no server-side routing, meaning:
      - Browser back/forward buttons cause blank pages or errors
      - Direct URL access to routes may fail
      - No middleware support for auth guards

  SOLUTION:
    Removed output: export from next.config.mjs
    Added security response headers (X-Frame-Options, X-XSS-Protection)
    Added compression: true for performance
    Static builds for Chrome Extension should use a separate build script

  FILE CHANGED: frontend/next.config.mjs


================================================================================
OPTIMIZATION 2: ROUTE MIDDLEWARE
================================================================================

  PROBLEM:
    No server-side middleware existed. Protected routes relied entirely
    on client-side JavaScript which:
      - Shows a flash of content before redirect
      - Can be bypassed by disabling JavaScript
      - Doesn't set proper cache-control headers

  SOLUTION:
    Created middleware.ts at frontend root
    Sets Cache-Control: no-store for proper back/forward behavior
    Sets X-Robots-Tag to protect dashboard from search crawlers
    Lets API routes and static files pass through untouched

  FILE CREATED: frontend/middleware.ts

OPTIMIZATION 6: SPA NAVIGATION (Link vs <a>)
================================================================================

  PROBLEM:
    Login page (app/login/page.tsx) used raw HTML <a href> tags instead
    of Next.js <Link> components:
      <a href="/login/candidate/">   (causes full page reload)
      <a href="/login/hr/">          (causes full page reload)
      <a href="/signup/">            (causes full page reload)

    This causes:
      - Full page reloads on every click (flash to white)
      - Browser history gets polluted with full reloads
      - Back/forward behavior becomes inconsistent
      - SPA state (auth context, theme) gets lost

  SOLUTION:
    Replaced all <a href> with Next.js <Link href>:
      <Link href="/login/candidate">   (client-side navigation)
      <Link href="/login/hr">           (client-side navigation)
      <Link href="/signup">             (client-side navigation)
    Also removed trailing slashes (cleaner URLs)

  FILE CHANGED: frontend/app/login/page.tsx

OPTIMIZATION 9: SECURITY HEADERS
================================================================================

  Added via next.config.mjs headers() function:
    X-DNS-Prefetch-Control: on        (faster DNS resolution)
    X-Content-Type-Options: nosniff   (prevent MIME sniffing)
    X-Frame-Options: SAMEORIGIN       (prevent clickjacking)
    X-XSS-Protection: 1; mode=block   (XSS filter)
    Referrer-Policy: strict-origin-when-cross-origin

  Added via middleware.ts:
    Cache-Control: no-store, must-revalidate (proper back/forward)
    X-Robots-Tag: noindex, nofollow (dashboard protection)

  FILE CHANGED: frontend/next.config.mjs
  FILE CREATED: frontend/middleware.ts
